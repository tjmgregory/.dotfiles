Plan: ship the editable-doc experiment

Draft v0.3 — replacing markdown chat with structured, mutable HTML.

OwnerTheo StatusIn review TargetThis week

Goal

Let the human edit any part of the assistant's response in-place — titles, bullets, table cells, even this sentence — without round-tripping a new prompt for everyasdf tiny correction.

Why HTML beats markdown here

  • Rich structure (tables, nested lists, pills) renders natively.
  • Every node is addressable, so the human can mutate it direct
  • One line of JS — document.designMode = "on" — makes the whole document a text field.

Open questions

  1. How do we send the user's edits back to the model as the next turn?
  2. Do we diff against the original, or just resend the full edited HTML?
  3. What about images and embedded widgets — are those editable too?
